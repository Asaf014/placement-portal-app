PRODUCTION DEPLOYMENT GUIDE

PRODUCTION REQUIREMENTS:
========================
- Ubuntu Server 20.04+ or similar Linux
- Python 3.8+ with pip
- Node.js 14+
- Redis Server
- Gunicorn or uWSGI for Python WSGI
- Nginx for reverse proxy
- SSL Certificate (Let's Encrypt recommended)

BACKEND DEPLOYMENT:
===================

1. Install Dependencies:
   sudo apt-get update
   sudo apt-get install python3-pip python3-venv redis-server postgresql
   pip install -r requirements.txt

2. Create Virtual Environment:
   python3 -m venv venv
   source venv/bin/activate

3. Install Gunicorn:
   pip install gunicorn

4. Set Environment Variables:
   cp backend/.env.example backend/.env
   nano backend/.env
   
   Change:
   FLASK_ENV=production
   SECRET_KEY=<generate-secure-key>
   JWT_SECRET_KEY=<generate-secure-key>
   SQLALCHEMY_DATABASE_URI=postgresql://user:password@localhost/dbname

5. Create Systemd Service (backend):
   sudo nano /etc/systemd/system/placement-portal.service
   
   [Unit]
   Description=Placement Portal Backend
   After=network.target
   
   [Service]
   User=www-data
   WorkingDirectory=/var/www/placement-portal/backend
   Environment="PATH=/var/www/placement-portal/backend/venv/bin"
   ExecStart=/var/www/placement-portal/backend/venv/bin/gunicorn --workers 4 --bind 0.0.0.0:5000 app:app
   Restart=always
   
   [Install]
   WantedBy=multi-user.target

6. Start Service:
   sudo systemctl daemon-reload
   sudo systemctl enable placement-portal.service
   sudo systemctl start placement-portal.service

FRONTEND DEPLOYMENT:
====================

1. Build Frontend:
   cd frontend
   npm run build

2. Deploy dist/ folder to web server (Nginx)

3. Nginx Configuration:
   sudo nano /etc/nginx/sites-available/placement-portal
   
   server {
       listen 80;
       server_name yourdomain.com;
       
       location / {
           root /var/www/placement-portal/frontend/dist;
           try_files $uri $uri/ /index.html;
       }
       
       location /auth {
           proxy_pass http://localhost:5000;
           proxy_set_header Host $host;
       }
       
       location /admin {
           proxy_pass http://localhost:5000;
           proxy_set_header Host $host;
       }
       
       location /company {
           proxy_pass http://localhost:5000;
           proxy_set_header Host $host;
       }
       
       location /student {
           proxy_pass http://localhost:5000;
           proxy_set_header Host $host;
       }
   }

4. Enable Site:
   sudo ln -s /etc/nginx/sites-available/placement-portal /etc/nginx/sites-enabled/
   sudo nginx -t
   sudo systemctl restart nginx

CELERY DEPLOYMENT:
==================

1. Create Celery Worker Service:
   sudo nano /etc/systemd/system/placement-portal-celery.service
   
   [Unit]
   Description=Placement Portal Celery Worker
   After=network.target redis-server.service
   
   [Service]
   User=www-data
   WorkingDirectory=/var/www/placement-portal/backend
   Environment="PATH=/var/www/placement-portal/backend/venv/bin"
   ExecStart=/var/www/placement-portal/backend/venv/bin/celery -A tasks.celery_tasks worker --loglevel=info
   Restart=always
   
   [Install]
   WantedBy=multi-user.target

2. Create Celery Beat Service:
   sudo nano /etc/systemd/system/placement-portal-beat.service
   
   [Unit]
   Description=Placement Portal Celery Beat
   After=network.target redis-server.service
   
   [Service]
   User=www-data
   WorkingDirectory=/var/www/placement-portal/backend
   Environment="PATH=/var/www/placement-portal/backend/venv/bin"
   ExecStart=/var/www/placement-portal/backend/venv/bin/celery -A tasks.celery_tasks beat --loglevel=info
   Restart=always
   
   [Install]
   WantedBy=multi-user.target

3. Start Services:
   sudo systemctl enable placement-portal-celery.service
   sudo systemctl start placement-portal-celery.service
   sudo systemctl enable placement-portal-beat.service
   sudo systemctl start placement-portal-beat.service

SSL/HTTPS SETUP:
================

1. Install Certbot:
   sudo apt-get install certbot python3-certbot-nginx

2. Generate Certificate:
   sudo certbot --nginx -d yourdomain.com

3. Nginx will auto-update with HTTPS

DATABASE SETUP:
===============

For PostgreSQL (recommended for production):

1. Install PostgreSQL:
   sudo apt-get install postgresql postgresql-contrib

2. Create Database:
   sudo -u postgres psql
   CREATE DATABASE placement_portal;
   CREATE USER ppadmin WITH PASSWORD 'secure_password';
   ALTER ROLE ppadmin SET client_encoding TO 'utf8';
   ALTER ROLE ppadmin SET default_transaction_isolation TO 'read committed';
   ALTER ROLE ppadmin SET default_transaction_deferrable TO on;
   GRANT ALL PRIVILEGES ON DATABASE placement_portal TO ppadmin;

3. Update .env:
   SQLALCHEMY_DATABASE_URI=postgresql://ppadmin:secure_password@localhost/placement_portal

REDIS SECURITY:
===============

1. Configure Redis:
   sudo nano /etc/redis/redis.conf
   
   requirepass <strong_password>

2. Update .env:
   REDIS_URL=redis://:password@localhost:6379/0

BACKUP STRATEGY:
================

Daily Database Backups:
   sudo nano /usr/local/bin/backup-portal.sh
   
   #!/bin/bash
   pg_dump -U ppadmin placement_portal > /backups/portal-$(date +%Y%m%d).sql
   
   Make executable:
   sudo chmod +x /usr/local/bin/backup-portal.sh

Add to Crontab:
   sudo crontab -e
   0 2 * * * /usr/local/bin/backup-portal.sh

MONITORING:
===========

1. Setup Logging:
   Configure logging in app.py for production
   Save logs to /var/log/placement-portal/

2. Monitor Services:
   systemctl status placement-portal
   systemctl status placement-portal-celery
   systemctl status placement-portal-beat
   systemctl status redis-server

3. Application Health Check:
   curl http://localhost:5000/health

PERFORMANCE TUNING:
===================

1. Gunicorn Workers:
   workers = (2 * CPU_count) + 1
   Example: For 4 CPUs: workers = 9

2. Connection Pooling:
   SQLALCHEMY_ENGINE_OPTIONS = {
       'pool_size': 10,
       'pool_recycle': 3600,
   }

3. Redis Settings:
   Set maxmemory and eviction policy

ROLLBACK PROCEDURE:
===================

1. Backup Current Version
2. Restore Previous Database Backup
3. Rollback Application Code
4. Restart Services

TROUBLESHOOTING:
================

Service won't start:
   sudo journalctl -u placement-portal -n 20

Permission denied:
   sudo chown -R www-data:www-data /var/www/placement-portal

Port already in use:
   sudo lsof -i :5000
   sudo kill -9 <PID>

Database connection error:
   Check PostgreSQL service: sudo systemctl status postgresql
   Verify connection string in .env

   limiter = Limiter(app, key_func=get_remote_address)
   
   @app.route('/auth/login', methods=['POST'])
   @limiter.limit("5 per minute")
   def login():
       ...

MAINTENANCE WINDOW:
===================

Schedule maintenance during off-peak hours:
1. Take app offline
2. Backup database
3. Apply updates
4. Test functionality
5. Bring app online

Keep maintenance window < 30 minutes for better user experience.
