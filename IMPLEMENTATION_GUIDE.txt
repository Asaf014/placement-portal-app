PLACEMENT PORTAL APPLICATION - IMPLEMENTATION GUIDE

PROJECT OVERVIEW
================
The Placement Portal Application is a complete web-based system designed to manage campus recruitment activities. It supports three user roles: Admin (Institute), Company, and Student. The application is built using modern web technologies and follows industry best practices.

TECHNOLOGY STACK
================
Backend:
- Flask 2.3.3 - Lightweight Python web framework
- SQLAlchemy - ORM for database operations
- Flask-JWT-Extended - JWT token-based authentication
- Redis - In-memory caching layer
- Celery - Distributed task queue for background jobs
- SQLite - Lightweight file-based database

Frontend:
- Vue.js 3 - Progressive JavaScript framework
- Vue Router - SPA routing
- Axios - HTTP client
- Bootstrap 5 - CSS framework with responsive design

ARCHITECTURE OVERVIEW
====================
The application follows a modular, layered architecture:

Layer 1: Database Layer (models/)
- User: Core user model with role-based differentiation
- Company: Company profile and details
- Student: Student profile and academic information
- PlacementDrive: Job postings and recruitment events
- Application: Student applications to drives
- ActivityLog: Audit trail of user actions
- Notification: User notification system

Layer 2: Service Layer (services/)
- UserService: User management and authentication
- CompanyService: Company profile management
- StudentService: Student profile management
- PlacementDriveService: Drive creation and management
- ApplicationService: Application handling

Layer 3: Route Layer (routes/)
- auth.py: Authentication endpoints
- admin.py: Admin-specific operations
- company.py: Company-specific operations
- student.py: Student-specific operations

Layer 4: Task Layer (tasks/)
- Celery tasks for background jobs
- Email notifications
- CSV export functionality
- Scheduled reminders

Layer 5: Utility Layer (utils/)
- Authentication helpers (JWT, password hashing)
- Caching utilities (Redis integration)
- Email sending utilities
- Common helper functions

SETUP INSTRUCTIONS
==================

PREREQUISITES:
- Python 3.8+
- Node.js 14+
- Redis Server
- Git

QUICK START (WINDOWS):

1. Extract the project to a folder
2. Open Command Prompt in the project root
3. Run: setup.bat
4. Follow the on-screen instructions

MANUAL SETUP:

Backend Setup:
1. cd backend
2. python -m venv venv
3. venv\Scripts\activate
4. pip install -r requirements.txt
5. python init_db.py
6. python app.py

Frontend Setup:
1. cd frontend
2. npm install
3. npm run dev

Redis Setup:
- Download and install Redis from https://github.com/microsoftarchive/redis/releases
- Run: redis-server

Celery Worker:
1. cd backend
2. venv\Scripts\activate
3. celery -A tasks.celery_tasks worker --loglevel=info

BACKGROUND JOBS
================

1. Daily Reminders (Scheduled):
   - Sends email reminders 1, 3, and 7 days before application deadline
   - Targets students who haven't applied yet

2. Monthly Reports (Scheduled):
   - Generates HTML report with placement statistics
   - Sent to admin email on first day of month
   - Includes: drives conducted, students applied, students selected

3. CSV Export (User-Triggered):
   - Students can export their application history
   - Sent via email in CSV format
   - Includes: student ID, company, drive, status, dates

4. Interview Reminders (Scheduled):
   - Sends reminders to selected candidates
   - Includes interview date and company details

AUTHENTICATION FLOW
===================

1. User registers with email and password
2. Password is hashed using Werkzeug security
3. User logs in with credentials
4. Backend generates JWT token (24-hour expiry)
5. Token stored in localStorage on frontend
6. All API requests include Authorization header
7. Backend validates token before processing requests
8. Token automatically includes user_id and role

COMMON ISSUES & SOLUTIONS
==========================

Issue: Redis connection refused
Solution: Ensure redis-server is running on localhost:6379

Issue: Database is locked
Solution: Delete placement_portal.db and reinitialize

Issue: CORS errors
Solution: Verify backend running on 5000 and frontend on 8080

Issue: Static files not loading
Solution: Build frontend app and ensure public/index.html exists

Issue: Email not sending
Solution: Configure MAIL_USERNAME and MAIL_PASSWORD in .env
